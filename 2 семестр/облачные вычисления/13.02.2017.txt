Для зачёта:
- как можно использовать облачные технологии в вашем дипломе?

================================================

Идея облачных вычислений появилась в 60-х годах.

client
^
|
| DISTRIBUTED SYSTEMS           DEVICES AND CLOUD COMPUTING
|
|
| CLIENT
| SERVER     BROWSER
|            SERVER
| MAINFRAME
|
--------------------------------------------> server

- распределённые системы просто запущены на многих компьютерах
- мэйнфрейм - большая машина для сложных вычислений; клиенты работают с ним через терминалы, они дешёвые (по сути - "телевизор с клавиатурой")

================================================

Концепция облаков

     Cloud
|            |
|            |
|            |
|            |
Client    Client

- клиент должен работать с облаком через интернет
- клиентов довольно много
- клиенты работают в разное время

Почему бы не совмещать все их вычисления на одной машине? Можно было бы разделять вычислительные ресурсы между клиентами. Для мейнфреймов сложность была в том, что все вычисления были последовательными. Затем появилась виртуализация (на одном компьютере могут быть запущены разные ОС, и разные пользователи могут с ними одновременно работать).

Характеристики:
- эластичность
- доступность
- масштабирование: вертикальное и горизонтальное (добавление новых серверов)

Модели облачных вычислений:
- IAAS (Infrastructure as a service)
- PAAS (Platform as a service)
- SAAS (Software as a service)
