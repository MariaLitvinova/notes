3. Сроки
Время, необходимое для завершения проекта.
Работа имеет нелинейную сложность, и действовать нужно осторожно (оценив сложность задачи).

Для оценки обычно делают план-график проекта.

=============================================================

План-график

- берём все задачи и ставим на первую линию
- каждую задачу оцениваем независимо от других (получается, что все задачи начинаются одновременно)
- оцениваем длительность выполнения задачи (для маленьких задач можно легко перейти от оценки трудоёмкости к оценке времени)
- отслеживаем зависимости для каждой задачи

Зависимости:
1. Finish Start
Одна задача может быть начата не раньше, чем закончена другая.
Пример: нельзя начать класть крышу, не поставив стены.
2. Start Start(?)
Задача B не может начаться раньше, чем началась задача А.
Пример: должна одновременно начаться поставка материалов и их использование.
3. Finish Finish
Задача В заканчивается не ранее, чем закончилась задача А.
Пример: аренда площадки должна закончиться не раньше, чем мы разобрали стенды на выставке.
4. Start Finish
Задача В может закончиться не ранее, чем задача А началась.
Пример: ???

На самом деле, самая частая зависимость -- Finish Start. Иногда ещё Finish Finish.

Замечание: 
- можно делать диаграммы в Microsoft Project
- можно в его оперсорс аналогах

Картинка в итоге окажется самый оптимистичный вариант завершения проекта (где можно, задачи будут параллельны, пауз не будет и т.д.). 
Для каждой задачи можно определить, кто будет её делать.
- распределить позиции (рабочее время сотрудников)
- какие-то дедлайны есть у проекта изначально, и под эти сроки надо при оценке подстраиваться
- проверить доступность ресурсов (есть люди, которых мы хотим задействовать, есть возможность отпускать их в отпуск/на сессию -- зная особенность сотрудников, уточняем оценку)
- выделяем важные вехи (1 версия, 2 версия,.../закончен модуль 1, закончен модуль 2,...); их тоже имеет смысл включить в план

4. Ещё надо оценивать стоимость
Когда мы знаем трудоёмкость и сроки, можем оценить, какие финансовые затраты необходимы, чтобы проект состоялся.

Оценка:
- fixed price
- T&M

Fixed Price:
- цена = стоимость ресурсов + вознаграждение за риск
- вознаграждение за риск = 30%-50% от стоимости (на то, что что-то случится)
- оплата привязана к фазам в случае длительных проектов, производится после приёмки работ заказчиком (надо же платить деньги сотрудникам)
- возможен аванс до начала работ
- значительная часть суммы выплачивается после окончания работ

T&M:
- цена = к-во человек * ставка/час * к-во принятых часов
- оплата производится каждый месяц или поэтапно
- возможна оплата допрасходов по договорённости

Управление персоналом

- что это?

- как увольнять сотрудников

Управление персоналом -- деятельность, связанная с работой с сотрудниками. Некоторые практики обязательны, некоторые зависят от компании.
- кадровый учёт (обязательная практика)
- подбор сотрудников (HR)
- формирование команды
- обучение 
- аттестация
- хантинг
- корпоротивы
- поощрения и наказания
- планирование отпусков
- ...


Типичный состав команды

- руководитель
Отвечает за успех проекта и различные деятельности в проекте
- архитектор
Проектирует ПО, выбирает технологии, выделяет структурные блоки
- аналитик
Переводит обе стороны во взаимодействии заказчика и исполнителя; извлекает информацию из заказчика
- технический лидер
Ставит задачи и контролирует их выполнение (?)
- ведущий разработчик
Участвует в постановке задач младшим разработчикам
Может стать ведущим разработчиком, если проработал n лет на позиции разработчика
